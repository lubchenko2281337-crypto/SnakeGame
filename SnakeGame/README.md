# Змейка (WPF, MVVM)

Десктопная игра «Змейка» на **.NET 8** и **WPF** с паттерном **MVVM**.

## Запуск

Требуется **Windows** и **.NET SDK 8** (или новее с поддержкой `net8.0-windows`).

```bash
cd SnakeGame
dotnet run
```

Открытие в Visual Studio: файл решения в корне репозитория — `SnakeGame.slnx` (или открыть `SnakeGame/SnakeGame.csproj`).

Сборка Release:

```bash
dotnet build -c Release
```

Исполняемый файл: `bin/Release/net8.0-windows/SnakeGame.exe`.

## Управление

| Действие | Клавиши / UI |
|----------|----------------|
| Направление | Стрелки |
| Пауза | Пробел или кнопка «Пауза» / «Продолжить» |
| Старт | Кнопка «Старт» |
| Заново | Кнопка «Заново» |

Разворот на 180° запрещён (как в классической змейке).

## Режимы

- **Классика (стены)** — выход за границу поля = проигрыш.
- **Бесконечное поле** — выход с края переносит змейку на противоположную сторону.

## Архитектура

- **Models** — `Snake`, `Food`, `GameField`, `CellPosition`, перечисления `Direction`, `GameMode`, фаза игры `GamePhase`.
- **Services** — `GameEngine` (шаг игры, спавн еды), `CollisionService` (стены, самопересечение), `HighScoreStore` (рекорд в `%LocalAppData%\SnakeGame\highscore.json`).
- **ViewModels** — `GameViewModel` (таймер `DispatcherTimer`, команды, счёт), `SnakeSegmentDisplay` для координат сегментов в пикселях на `Canvas`.
- **Views** — `MainWindow.xaml`: разметка и привязки; направление стрелками обрабатывается в `OnPreviewKeyDown`, чтобы клавиши работали после клика по кнопкам; контекст и фокус задаются в code-behind окна.
- **Helpers** — `RelayCommand` для `ICommand`.

Игровой цикл: таймер → расчёт направления → `GameEngine.Tick` → обновление коллекции сегментов и позиции еды. Скорость постепенно растёт вместе с количеством съеденной еды.

## Расширяемость

Новые режимы и правила — через `GameMode` и ветки в `GameEngine` / `CollisionService`. Дополнительные сущности на поле (препятствия, бонусы) — отдельные модели и проверки в сервисе шага. Темы оформления — ресурсы и стили в XAML без изменения логики.
